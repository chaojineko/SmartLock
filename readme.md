# 端边云协同智能门禁系统（项目总览）

本项目是一个由微信小程序、ESP32 边缘设备和 STM32 执行控制器组成的智能门禁系统，支持二维码临时授权、多种本地身份验证方式以及设备联动控制。

系统按端边云协同思路实现：
- 云端负责授权生命周期管理（生成、校验、撤销、查询）。
- 手机端负责用户交互与授权下发。
- 边缘端负责二维码/人脸相关处理与联网校验。
- 控制端负责门锁执行与本地外设管理。

## 项目组成

### 1. 微信小程序与云函数（miniprogram-1）

主要职责：
- 生成二维码临时授权（支持时长模式和时间窗口模式）
- 查看当前生效中的授权并执行撤销。
- 通过 BLE 与门锁设备进行近场交互。

后端能力（云函数）：
- `createGrant`：创建授权并生成二维码数据。
- `verifyGrant`：校验授权有效性并执行消费逻辑。
- `listActiveGrants`：查询活跃授权。
- `revokeGrant`：撤销授权。

### 2. 边缘侧 AI 与联网模块（AI_Human_Face_Recognition）

主要职责：
- 处理摄像头输入并执行二维码识别/人脸相关流程。
- 在本地完成快速决策，同时通过网络与云端做授权校验。
- 与 STM32 通过串口联动，触发开锁/上锁行为。
- 支持 BLE 控制链路。

技术栈：
- ESP-IDF v5.4.1
- ESP-DL 相关组件
- FreeRTOS（ESP-IDF 内置）

### 3. 执行控制模块（projectfreertos）

主要职责：
- 本地多模态门锁控制：密码、指纹、RFID。
- OLED 菜单交互与矩阵键盘输入。
- 舵机控制与蜂鸣反馈。
- 接收边缘侧串口联动命令并执行。

技术栈：
- STM32F407VET6
- STM32CubeMX + HAL
- FreeRTOS
- Keil MDK

## 当前实现状态

已形成可演示的完整闭环：
- 手机端可创建临时授权并展示二维码。
- 设备侧可执行授权校验并联动门锁执行。
- STM32 侧可独立完成本地认证与开锁流程。

说明：
- 不同子工程各自独立构建与烧录。
- 联调需要保证串口连接、设备供电和网络环境正确。

## 快速联调流程

1. 先完成 STM32 工程编译与下载，确认本地菜单、舵机和认证流程正常。
2. 再完成 ESP32 工程编译与烧录，确认摄像头、网络与串口链路正常。
3. 最后在微信开发者工具部署小程序和云函数，确认授权创建与校验接口正常。
4. 进行端到端测试：创建授权 -> 设备识别/校验 -> 执行开锁 -> 状态回传。
